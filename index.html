SwiftUILab

A senior iOS engineer's path from UIKit to fluent SwiftUI & modern concurrency — one screen, one concept at a time.

This site is the companion to the SwiftUILab Xcode project. Read a module page, type the reference into Xcode, attempt the challenge in Challenge/, then compare against the solution reveal. Check the box in the sidebar to track progress (stored in localStorage).

How each module page is structured

SectionWhat it gives you
ConceptThe why and the runtime/semantic behavior, with UIKit contrasts where they sharpen the point.
PitfallsThe traps that bite senior engineers, not beginners.
ReferenceThe complete, correct code — also in the project under Reference/.
ChallengeA variation to implement from the Challenge/ boilerplate (TODO markers only).
Solution revealThe answer plus commentary on the key decisions.
Further readingWWDC sessions, Apple docs, community deep-dives.

Status

Built Module 1 — State System. Modules 2–10 are scaffolded in the app (placeholder screens) and stubbed here; each fills in during its own session.

Running things

The app: open SwiftUILab/SwiftUILab.xcodeproj in Xcode 26+ and run on an iOS 26 simulator.

This site: open guide/index.html directly, or from the repo root run npm run dev (a tiny static server) and visit the printed URL.
