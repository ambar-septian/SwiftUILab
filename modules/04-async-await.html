Module 04

Async / Await

Suspension points, the Task lifecycle, cooperative cancellation, async sequences, and bridging callback APIs with continuations — plus the all-important .task vs Task { } distinction in views.

Concept

Suspension points

An await marks a suspension point: the function may pause there, releasing the thread back to the cooperative thread pool, and resume later — possibly on a different thread. Nothing blocks. This is the mental shift from GCD: you don't hop queues to avoid blocking, you await and the runtime reuses the thread for other work. Between two awaits your code runs synchronously and atomically with respect to its actor.

func fetch() async throws -> [Article] {
    try await Task.sleep(for: .milliseconds(600))   // suspension point — thread is freed
    return articles
}

Task lifecycle & priorities

A Task is a unit of asynchronous work. An unstructured task (Task { }) starts immediately, inherits the actor and priority of its origin, and runs independently — you must hold and cancel it yourself. A task has a priority (.high, .userInitiated, .utility, .background) that the scheduler uses; it's a hint, not a guarantee.

.task vs Task { } in views

This is the most common source of leaked work in SwiftUI:

.task { }Task { }
Tied to view lifetimeYes — auto-cancelled on disappearNo — outlives the view
Restart on identity change.task(id:) re-runsManual
Who cancels itThe frameworkYou
Use forLoading tied to a viewFire-and-forget from a button tap

Use .task for "load while this screen is on-screen." Use Task { } inside a button action for a one-shot you don't need to cancel — but be aware it is not bound to the view.

Cooperative cancellation

Cancelling a task does not stop it — it only sets a flag. Long-running work must check Task.isCancelled (or call try Task.checkCancellation(), which throws CancellationError) and stop voluntarily. Most async stdlib calls (Task.sleep, URLSession) check for you and throw on cancellation.

for step in 1...100 {
    if Task.isCancelled { return }            // poll the flag
    try? await Task.sleep(for: .milliseconds(40))
    progress = step
}

AsyncSequence / AsyncStream

AsyncSequence is the async analogue of Sequence: you iterate it with for await. AsyncStream is the easiest way to build one — bridge any push-based source (a delegate, a timer, a Combine publisher) by calling continuation.yield(_:), then continuation.finish(). Set onTermination to clean up when the consumer stops or is cancelled.

AsyncStream<Int> { continuation in
    let task = Task { for n in stride(from: 5, through: 0, by: -1) { continuation.yield(n) } ; continuation.finish() }
    continuation.onTermination = { _ in task.cancel() }   // propagate cancellation upstream
}

withCheckedContinuation bridging

To adapt a callback API into async, wrap it in withCheckedContinuation (or the throwing variant) and resume the continuation exactly once. The checked form adds runtime assertions that catch the two classic bugs: resuming twice (crash) or never (a task that hangs forever).

func geocode(_ q: String) async throws -> Int {
    try await withCheckedThrowingContinuation { continuation in
        LegacyGeocoder.lookup(q) { result in continuation.resume(with: result) }
    }
}

Common pitfalls & gotchas

  • Task { } in a view body or onAppear leaks. It isn't tied to the view; if the view disappears the work keeps running and may update detached state. Prefer .task.
  • Forgetting cancellation is cooperative. A tight compute loop with no Task.isCancelled check will run to completion even after .cancel(). Add checkpoints.
  • Resuming a continuation zero or two times. Zero → the awaiting task hangs forever; two → crash. Every path through the callback must resume exactly once. Use the checked variant during development.
  • Swallowing CancellationError as a real error. When a .task is cancelled, the in-flight call throws CancellationError. Catch it separately and leave state untouched, rather than showing a spurious "failed" UI.
  • Updating UI off the main actor. Work after an await may resume off the main thread. In a SwiftUI view (main-actor isolated) you're fine; in a plain helper, hop back with @MainActor or await MainActor.run.
  • AsyncStream without onTermination. If you don't cancel the producing task when the consumer stops, it runs forever — a subtle leak.

// Typed load phase drives the view, loaded via .task(id:)
enum LoadPhase<Value: Equatable>: Equatable { case idle, loading, loaded(Value), failed(String) }

.task(id: useFailing) {
    phase = .loading
    do { phase = .loaded(try await service.fetch()) }
    catch is CancellationError { /* view went away — ignore */ }
    catch { phase = .failed(error.localizedDescription) }
}

// Cooperative cancellation: stored Task + isCancelled checks
work = Task {
    for step in 1...100 {
        if Task.isCancelled { status = "Cancelled"; return }
        try? await Task.sleep(for: .milliseconds(40)); progress = step
    }
}
// Cancel button: work?.cancel()

// Consuming an AsyncStream in a view
.task(id: restartToken) { for await n in Countdown.stream(from: 5) { current = n } }

// Bridging a callback API
func geocode(_ q: String) async throws -> Int {
    try await withCheckedThrowingContinuation { c in
        LegacyGeocoder.lookup(q) { c.resume(with: $0) }
    }
}

Solution reveal

Show the solution & commentary 
// 1. Loading
@State private var phase: LoadPhase<[Article]> = .idle
var body: some View {
    Group {
        switch phase {
        case .idle, .loading: ProgressView()
        case .loaded(let a):  ForEach(a) { Text($0.title) }
        case .failed(let m):  Text(m).foregroundStyle(.red)
        }
    }
    .task {
        phase = .loading
        do { phase = .loaded(try await LiveArticleService().fetch()) }
        catch is CancellationError {}
        catch { phase = .failed(error.localizedDescription) }
    }
}

// 2. Cancellation
@State private var work: Task<Void, Never>?
func start() {
    work = Task {
        for i in 1...100 {
            if Task.isCancelled { return }
            try? await Task.sleep(for: .milliseconds(40)); progress = i
        }
        work = nil
    }
}
// Cancel: work?.cancel()

// 3. Stream
@State private var current: Int?
.task { for await n in Countdown.stream(from: 5) { current = n } }

// 4. Your own bridge
func myGeocode(_ q: String) async throws -> Int {
    try await withCheckedThrowingContinuation { c in
        LegacyGeocoder.lookup(q) { c.resume(with: $0) }   // resume exactly once
    }
}

Key decisions:

  • Catch CancellationError before the generic catch so a normal disappearance doesn't render as an error.
  • Storing the Task in @State is what lets a separate button cancel it; clearing it to nil on completion gates the Start/Cancel buttons.
  • The stream's onTermination (in Countdown) cancels the producer when .task tears down — no leak.
  • Using resume(with:) forwards the Result directly, so success and failure both route through one call — easy to keep "exactly once."
